# SPIC-PPT · 符合国家电投规范的演示文稿制作 Skill

> 给 Claude Code / Claude Agent 用的 PPT 制作 skill
> 把"已成型的研究报告/大纲"快速装配成符合国家电投VI规范的演示文稿
> 适用：国家电投体系内各单位日常汇报、研究成果、专题分析等PPT场景

---

## 核心定位

**PPT 工作流是视觉装配流水线，不是内容生成器。**

```
研究阶段（你做）→ 形成报告/大纲 → SPIC-PPT skill（AI 装配）→ 定稿 PDF
```

- 你的核心价值：研究 + 判断（已经做完）
- AI 的核心价值：抽取 + 减法 + 装配
- 目标时长：**40-60 分钟** 完成一份 20-30 页的标准汇报

---

## 快速开始

### 在 Claude Code 中使用

1. 把本仓库克隆到 `~/.claude/skills/SPIC-PPT/`
2. 启动 Claude Code
3. 跟 Claude 说："**按 SPIC-PPT 工作流做一份 XX 汇报，主输入在 [路径]**"
4. 回答 4 个约束问题（用途/时长/听众/风格）
5. 审定 AI 生成的"结构化抽取"
6. 等 AI 装配 HTML、导出 PDF

### 你需要带什么进来

| 必带 | 可带 | 不要带 |
|---|---|---|
| 完整成稿（报告/大纲） | 取舍指引 | 已经"嚼过"的核心论点 |
| 使用约束（4个问题） | 必保留原文 | 大纲/讲稿/视觉指令 |

详见 [task-input-checklist.md](task-input-checklist.md)

## 工作流八阶段（节选）

| 阶段 | AI 主导 | 用户主导 |
|---|---|---|
| 1 接收任务 + 接收成稿 | 问 4 个约束 | 提供成稿、答约束 |
| 2 读懂成稿 + 权威文本补充 | 通读、识别三类内容、抓权威原文 | 提供内部材料 |
| 3 结构化抽取 + 版式建议 | 抽核心、推荐版式、做减法 | 审定取舍 |
| 4 技术选型 | 默认 HTML+CSS | — |
| 5 HTML 装配 | 套标准模板 | — |
| 6 质量校验 | 子智能体引文校验 | 政治合规审定 |
| 7 PDF 导出 + 反向验证 | Chrome headless + PyMuPDF 检查 | — |
| 8 归档 | 文件组织 | — |

详见 [workflow-sop.md](workflow-sop.md)

## 四条铁律

1. **LOGO 绝不自制**——党徽、企业 LOGO 必须从 `shared/logos/` 调用
2. **党政文件引用必须从官方源抓取**——不能凭记忆、不能用二手解读
3. **AI 做减法不做加法**——保留用户的核心表述原文
4. **进入 PPT 工作流后不改研究内容**——发现研究问题→退回研究阶段
