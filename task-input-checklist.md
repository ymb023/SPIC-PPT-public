# 任务启动输入清单 · v2

> 在进入工作流SOP之前，你需要带什么进来
> v2 修订核心：承认"研究产物（报告/大纲）就是 PPT 的内容源"

---

## 工作流的真实定位

```
┌──────────────────────┐         ┌──────────────────────┐
│  研究阶段（你做）       │   →     │  PPT 工作流（AI 装配） │
│  full-research     │         │                       │
│  quick-research       │         │  把成稿视觉化为 PPT     │
│  形成报告/大纲         │         │                       │
└──────────────────────┘         └──────────────────────┘
        上游产物                          下游环节
```

**核心判断**：PPT 工作流是**视觉装配流水线**，不是**内容生成器**。

- 你的研究产物（报告/大纲）= PPT 的内容源
- AI 的工作 = 读懂成稿 → 结构化抽取 → 版式适配 → 标准模板装配
- 你不需要在这一阶段重新做内容工作

---

## 你应该带什么进来

### 1. 一份"主输入"（必须）

**最优先**：你已经形成的成稿。

形态可以是：
- 完整的研究报告 `.docx` / `.md` / `.pdf`
- 完整的大纲 `.md`
- 你写好的文字稿

**这一份就够了**——不需要再拆解成"核心论点+关键数据+想法清单"。AI 应该自己读你的成稿。

### 2. 使用约束（必须）

只需 3 件事：

| 项 | 例 |
|---|---|
| **听众 + 场景** | "给集团董事长 + 院领导 + 部门一把手，专题汇报会" |
| **时长** | "45 分钟" |
| **风格** | "标准汇报" / "学术分享" / "存底备查" / "对外发布" |

### 3. 哪些必讲、哪些可删（可选但有用）

报告往往比 PPT 容得下的内容多 2-3 倍，所以告诉 AI 哪些是核心、哪些可舍：

```
- 必讲：[报告第 X 章 / 第 Y 节]
- 可略：[第 Z 章 / 时间不够可以删]
- 不在 PPT 里讲：[附录 / 方法论细节 / 参考文献]
```

如果你不指定，AI 会按自己的判断做减法——一般也能讲清楚，但可能错过你最想突出的点。

### 4. 必须保留原文的部分（可选但重要）

如果报告里有**核心论断或差异化表述**是你的原创/亮点，明确说"这几句必须原文保留"。

例：
- "'电力正在从基础保障品转变为决定产业布局、算力部署和国家竞争力的核心底座'这句必须原文用"
- "'三个转变'框架不能改"

## 你不需要做的事

| 不需要做 | 原因 |
|---|---|
| 再画大纲 | 你的报告就是大纲，AI 应该读原文 |
| 再抽"核心论点+关键数据" | 这是 AI 的工作（结构化抽取） |
| 把报告改写成"想覆盖的内容点清单" | 你的报告本身就是结构化的，重做是浪费 |
| 提前删减报告内容 | 让 AI 做减法，你做减法可能误删 |
| 标注每页放什么版式 | 版式选择是 AI 基于内容性质做的 |
| 指定页数 | 时长决定页数 |
| 改格式（字号配色） | 标准模板已经定义 |

---

## AI 在装配 PPT 时应该做的（修订）

接到带成稿的任务时，AI 应该：

### 第一步：读懂成稿（不是问"你要讲什么"）

**❌ 错误做法**：让用户重新提炼论点
**✅ 正确做法**：自己读完整成稿，输出"我读到的核心结构"让用户校对

### 第二步：结构化抽取

从成稿里识别三类内容：
- **判断/论点**：要做成PPT的核心要素
- **支撑数据/引用**：要用绿色虚线框或者整段染红
- **过渡内容**：通常不进PPT（"综上所述""下面我们来看"这类）

### 第三步：版式适配建议

按内容性质建议每节用哪种版式（标准模板提供的10种版式中选）：
- 多个并列要点 → 三栏卡片 / 编号点列
- 数据对比 → 双栏图表+表格 / 数据指标墙
- 时间序列 → 时间线
- 单一核心趋势 → 全宽图表
- 结构性变化 → 双栏文字结论

提出建议后**等用户审定**，再装配 HTML。

### 第四步：做减法

把成稿浓缩为 PPT 适合的密度。一页 PPT 平均承载量：
- 论点 1-3 条
- 数据/引文 2-4 条
- 字数 ≤ 200 字（含标点）

**保留你的核心表述原文**——重要判断要尽量保留你的措辞，不要替你改写。

### 第五步：标准模板装配

套用 `template/`，不要发明新版式。

## 修订后的工作流八阶段

| 阶段 | v1（错） | v2（修订） |
|---|---|---|
| 1 | 接收任务 + 问6个问题 | **接收任务 + 接收成稿** |
| 2 | 素材收集（让用户给原料） | **AI 读懂成稿**（含权威文本补充） |
| 3 | **大纲编制**（V1→V2→V3） | **结构化抽取 + 版式适配建议**（让用户审定） |
| 4 | 技术选型 | （不变） |
| 5 | HTML 制作 | **HTML 装配** |
| 6 | 质量校验 | （不变） |
| 7 | PDF 导出 | （不变） |
| 8 | 归档 | （不变） |

**核心变化在 1-3 阶段**：从"AI 帮你想"转变为"AI 帮你装配"，**减少了内容生成，增加了内容抽取**。

## 标准启动话术

### 推荐版

```
我要做一份 PPT，主输入在 [路径]。
听众：[谁，什么级别]
场景：[现场/存底/审核]
时长：[XX 分钟]
风格：[标准汇报/学术分享/...]

请按工作流走，先读完主输入，然后告诉我你的结构抽取结果让我确认。
```

### 完整版（含取舍指引）

```
任务：把 [路径] 的研究报告做成 PPT

听众/场景/时长/风格：[同上]

取舍指引：
- 必讲：第 1、2、4 章
- 可略：第 3 章（时间不够可删）
- 不进 PPT：附录、参考文献

必须保留原文的：
- 第 2 章关于 "[XX 判断]" 的表述
- 第 4 章关于 "[YY 论断]" 的表述

约束：
- 引用党政文件必须用权威源原文
- 集团 X 数字不能讲
```

## 五个反模式（哪些不要做）

### 反模式 1：把报告"提前嚼一遍"再给 AI

❌ "我把报告的核心论点提取出来了，AI 你按这个做"

→ 你做了 AI 该做的工作，浪费时间，而且你的提取可能比 AI 的抽取还粗糙。

✅ 直接给完整报告，让 AI 自己抽取。

### 反模式 2："这份报告太长了我先删一删"

❌ 你预先删除自认为"不重要"的章节再交给 AI

→ AI 在浓缩成PPT时需要看全局做取舍判断。你先删可能误删关键支撑。

✅ 给完整版，用"取舍指引"告诉 AI 哪些可略。

### 反模式 3："你帮我看看，再加点别的内容"

❌ 让 AI 在你的报告基础上"扩充"内容

→ AI 加的内容大概率比不上你研究形成的原创判断，反而稀释核心。

✅ 让 AI 只做减法不做加法。如有不足，自己补研究产物。

### 反模式 4：分多次零碎扔素材

❌ 第一次扔报告，第二次扔领导讲话，第三次扔行业数据……每次让 AI 重新理解上下文

→ AI 每次接收都要重新建立上下文，效率低。

✅ 一次性把所有素材准备好，分级标注（主输入/参考/背景）一起给。

### 反模式 5：进了 PPT 工作流又想改研究内容

❌ "AI 你看这页讲的不对，应该是 XX..."（这是研究层面的修正）

→ 研究阶段没做透，到 PPT 阶段才发现，返工成本极高。

✅ 进入 PPT 工作流前，确认研究产物是定稿。如果发现研究有问题，**退回研究阶段重做**，不要在 PPT 工作流里改内容。

---

## 一个反向认识

**进入 PPT 工作流的本质，是"你已经研究完了，要把成果可视化"**：

- 你的核心价值是"研究 + 判断" → 这一阶段已经在 PPT 工作流之前完成
- AI 的核心价值是"装配 + 视觉" → 这一阶段才是 PPT 工作流要做的

**这个分工清晰后**：
- 你的时间不再花在"重新整理你已经整理好的内容"上
- AI 不再做"猜测你想说什么"的低效率工作
- PPT 工作流应该 30-60 分钟就能跑完（从接收成稿到导出PDF）
